# Xiaohongshu Tutorial Copy Guide

Use this reference when the user wants to share an AI + Zotero RSS/OPML workflow with beginners.

## Positioning

Lead with a beginner-friendly promise:

- "科研小白也能用：AI + Zotero 自动追踪最新文献"
- "别再每天手动刷 PubMed，让最新文献自动流进 Zotero"

Avoid making the post only about Zotero RSS. The distinctive hook is that AI generates the OPML subscription structure.

## Explain In Plain Language

- RSS: a source that keeps showing new articles.
- OPML: a batch subscription list for many RSS feeds.
- Zotero: the place where new articles appear and useful papers are saved.
- AI: converts field, journals, and keywords into an importable OPML.

## Beginner Prompt Template

```text
我想用 Zotero RSS 追踪最新文献。
我的研究方向是：____
我想关注的核心期刊有：____
我想追踪的关键词有：____
请你帮我生成 Zotero 可导入的 OPML 文件。
要求：用 PubMed RSS；订阅名称加 01、02、03 排序；每条 RSS 显示上限设为 100。
```

## Post Structure

1. Pain: manually checking PubMed and journal websites is scattered and easy to forget.
2. Solution: ask AI to build a Zotero OPML subscription list.
3. Plain explanation: OPML is a batch RSS list.
4. Concrete example: numbered feeds such as `01-11` journals and `12` top-journal keyword radar.
5. Action: import the OPML into Zotero, then scan RSS and save important papers.
6. Close: AI does not replace reading; it builds the information entrance.

## Emoji Style

Use emoji in social copy, not inside formal OPML feed names:

- 😭 or 🥲 for pain points.
- ✅ for problem lists.
- 🤖 when introducing AI.
- 📦 when explaining OPML.
- 📚 when discussing Zotero library saving.
- ✨ or 🌟 for the final benefit and save prompt.
